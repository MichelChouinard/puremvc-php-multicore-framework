  27. /**
  28.  * A base <b>ICommand</b> implementation that executes other <b>ICommand</b>s.
  29.  *
  30.  * A <b>MacroCommand</b> maintains an list of
  31.  * <b>ICommand</b> Class references called <i>SubCommands</i>.
  32.  *
  33.  * When <b>execute</b> is called, the <b>MacroCommand</b>
  34.  * instantiates and calls <b>execute</b> on each of its <i>SubCommands</i> turn.
  35.  * Each <i>SubCommand</i> will be passed a reference to the original
  36.  * <b>INotification</b> that was passed to the <b>MacroCommand</b>'s
  37.  * <b>execute</b> method.
  38.  *
  39.  * Unlike <b>SimpleCommand</b>, your subclass
  40.  * should not override <b>execute</b>, but instead, should
  41.  * override the <b>initializeMacroCommand</b> method,
  42.  * calling <b>addSubCommand</b> once for each <i>SubCommand</i>
  43.  * to be executed.
  44.  *
  45.  * @see Controller
  46.         org\puremvc\php\multicore\core\Controller.php
  47.  * @see Notification
  48.         org\puremvc\php\multicore\patterns\observer\Notification.php
  49.  * @see SimpleCommand
  50.         org\puremvc\php\multicore\patterns\command\SimpleCommand.php
  51.  * @package org.puremvc.php.multicore
  52.  */

  97.     protected function initializeMacroCommand()

  99.     }

  117.      *
  118.      * The <i>SubCommands</i> will be called in First In/First Out (FIFO)
